# my_project
Подбор одежды по погоде
Цель: создать сайт по подбору одежды по погоде, из домашней коллекции который имеет пользователь, и так же визуальный наброс одежды для виденья со стороны.
Почему и зачем: часто бывает, когда не знаешь, как одеться по погоде, чтобы чувствовать себя комфортно. С помощью сайта, пользователь сможет добавлять свои вещи на сайт, и мы сможем ему помочь подобрать нужный комплект вещей на сегодня (возможно даже на определенный период времени). Ну и чтобы все это выглядело комплексно, сделаем манекен, на котором можно примерять свою одежду, чтобы можно было посмотреть немного со стороны.
Суть работы проекта простая: фотографируем свою одежду(или берем картинку из магазина - интернета), заливаем ее на сайт, у картинки обрезается задний фон, и она подставляется на манекен (вручную или автоматически в зависимости от погоды).
Сайт подойдет для всех людей, что хотят выглядеть хорошо или быть одетым комфортно под погоду.
Подбирай одежду по:
•	температуре
•	дождь/снег
•	ветер (опционально)
•	Тип (футболка, джинсы, куртка), тёплость (1–5), сезон, влагостойкость

ЕСЛИ температура < 0°:
  верхний слой = пуховик (>4 тепла)
  средний слой = свитер (>3 тепла)
ЕСЛИ дождь И температура > 10°:
  верхний слой = куртка с влагостойкостью
ЕСЛИ ветер > 10 м/с:
  добавить аксессуар = шапка/шарф

Связи между ними (текстом)
https://www.notion.so/ER-e0a72012b7d64846b0cb539313edec51
1. User → Clothing (1 ко многим)
•	Один пользователь может добавить много вещей
•	Вещь принадлежит ровно одному пользователю
2. User → Outfit (1 ко многим)
•	Один пользователь может создать много образов
•	Образ принадлежит одному пользователю
3. Outfit → Clothing (многие ко многим через OutfitClothing)
•	Образ может содержать много вещей
•	Вещь может быть в многих образах
•	Поле layer_type определяет, куда надевать (нижний/средний/верхний/аксессуар)
4. User → Wardrobe (1 ко многим)
•	У пользователя может быть три коллекции (основная, избранное, примерочная)
•	Через WardrobeClothing связываем вещь с конкретной коллекцией
5. User → Friendship (1 ко многим, рекурсивно)
•	Пользователь может подписаться на многих
•	На пользователя могут подписаться многие
•	follower_id = кто подписался, following_id = на кого подписались
6. User → Like (1 ко многим)
•	Пользователь может лайкнуть много вещей
•	Вещь может получить много лайков
Упрощённая ER-диаграмма (текстовая)

User
  ├──< Clothing
  ├──< Outfit
  ├──< Wardrobe
  ├──< Friendship (как follower)
  ├──< Friendship (как following)
  └──< Like

Clothing
  ├──>─ User
  ├──>─ Like
  └──< OutfitClothing

Outfit
  ├──>─ User
  └──< OutfitClothing

OutfitClothing
  ├──>─ Outfit
  └──>─ Clothing

Wardrobe
  ├──>─ User
  └──< WardrobeClothing

WardrobeClothing
  ├──>─ Wardrobe
  └──>─ Clothing

Friendship
  ├──>─ User (follower)
  └──>─ User (following)

Like
  ├──>─ User
  └──>─ Clothing
Где:
•	< один ко многим (слева один, справа много)
•	> многие к одному
•	>─< многие ко многим (через промежуточную)
